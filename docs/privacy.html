Privacy Policy

Last updated: March 1, 2026

This Privacy Policy describes how the QuickBooks MCP Server application ("Application") handles data.

1. Overview

The Application is a self-hosted tool that runs entirely on your own infrastructure. It acts as a bridge between an MCP client (such as Claude Desktop) and the QuickBooks API.

2. Data Collection

The Application does not collect, store, or transmit any personal data to third parties. Specifically:

  • No analytics or tracking is performed.
  • No data is sent to any server other than the official QuickBooks API endpoints operated by Intuit.
  • No user data is logged or persisted beyond the runtime session.

3. QuickBooks API Data

When you use the Application, it makes API calls to QuickBooks on your behalf using credentials you provide. The data returned from QuickBooks is passed directly to your MCP client and is not stored, cached, or forwarded elsewhere.

4. Credentials

Your QuickBooks API credentials (Client ID, Client Secret, Refresh Token, Company ID) are provided by you via environment variables. These credentials are used solely to authenticate with the QuickBooks API and are never logged, transmitted, or shared.

5. Third-Party Services

The Application communicates only with:

  • QuickBooks API (quickbooks.api.intuit.com or sandbox-quickbooks.api.intuit.com) — to execute queries and API calls.
  • Intuit OAuth (oauth.platform.intuit.com) — to refresh access tokens.

6. Data Security

All communication with QuickBooks and Intuit OAuth endpoints is conducted over HTTPS. Since the Application runs on your own infrastructure, you are responsible for securing the environment in which it operates.

7. Changes to This Policy

This policy may be updated at any time. Continued use of the Application constitutes acceptance of the updated policy.

8. Contact

For questions about this privacy policy, please open an issue on the project's GitHub repository.
